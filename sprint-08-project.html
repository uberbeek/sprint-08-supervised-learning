The data look good overall. I will change the data type of the Tenure column from float to integer but the rest are appropriate.


I think RowNumber, CustomerId, and Surname are irrelevant for the purposes of modeling bank churn. The first two could contain useful information if they were assigned chronologically or by some other meaningful convention, but without more information about that, I will exclude them. Including arbitrary or irrelevant features in a model can lead to overfitting and distorted interpretations.


Five columns (Geography, Gender, HasCrCard, IsActiveMember, and Exited) are categorical with only a few values. Four columns (CreditScore, Age, Balance, and EstimatedSalary) are continuous, with approximately normal distributions for CreditScore and Age and a nearly uniform distribution for EstimatedSalary. Balance is also normally distributed except for an excess of zeros representing 36% of the sample. In some circumstances, I might split that into two columns (HasBalance and BalanceAmount) to separate the processes at work, but that probably is not necessary here. Two columns (Tenure and NumberOfProducts) are discrete and restricted in range to a dozen or fewer values but they are ordinal, so numeric labels are appropriate. None of the classification models I will use are particularly sensitive to normality assumptions, so these data should be fine.


The only missing values are in the Tenure column, representing 9% of the sample. I found no obvious pattern in the means of missing and non-missing rows. If the values are missing at random, filling them with the mean or median might make sense. If some accounts have no period of maturation, zeros might be more appropriate. If some accounts have no fixed deposits at all, filling in any value might be misleading. For example, owning no car is not equivalent to owning one with a zero-liter engine or even an average-sized engine. Without a better understanding of what Tenure actually means, I am hesitant to guess at the best imputation method. Given the large size of the sample, I think losing 9% is preferable to introducing questionable data, so I will drop it.


Categorical columns HasCrCard, IsActiveMember, and Exited are already in a binary (0 or 1) format, but Gender and Geography need to be encoded numerically. With just 2 and 3 categories each, One-Hot Encoding is suitable for both columns. Numerical columns CreditScore, Age, Tenure, Balance, NumOfProducts, and EstimatedSalary all need to be standardized. These transformations are necessary for the classification algorithms to appropriately weight the data.


The Exited column, which will be the target for modeling, has a considerable imbalance with 80% of values being 0 and the rest 1. That will require some adjustment during model optimization but not yet.


To summarize, I will drop from the training data the RowNumber, CustomerId, and Surname columns and the rows with missing values in the Tenure column. I will also change the Tenure column's data type from float to integer, apply One-Hot Encoding to Gender and Geography, and standardize CreditScore, Age, Tenure, Balance, NumOfProducts, and EstimatedSalary. With that done, I will split the data into train, validate, and test sets and proceed to modeling.

# Upsample the data

def upsample(features, target, repeat):
    features_zeros = features[target == 0]
    features_ones = features[target == 1]
    target_zeros = target[target == 0]
    target_ones = target[target == 1]

    features_upsampled = pd.concat([features_zeros] + [features_ones] * repeat)
    target_upsampled = pd.concat([target_zeros] + [target_ones] * repeat)

    features_upsampled, target_upsampled = shuffle(
        features_upsampled, target_upsampled, random_state=12345)

    return features_upsampled, target_upsampled

General Discussion






















The purpose of this project was to predict which customers are likely to leave Beta Bank based on demographic and banking data.


After checking the data, changing one data type, and dropping extraneous features and observations with missing values, I encoded and scaled features as appropriate. A fairly severe class imbalance in the target feature Exited (with 80% not exiting) led to poorly perfoming models. Addressing the imbalance by specifying the class_weight parameter and upsampling the data, I found that upsampling produced the best models. Among decision tree, random forest, and logistic regression models, random forest classifiers consistently performed the best. I optimized the number of estimators and max tree depth parameters to obtain an adequate random forest classifier with 25 estimators and a depth of 12. The final model performed well on the test set with an F1 score of 0.61 and AUC-ROC of 0.85, handily outperforming both a sanity check random model and the project F1 threshold of 0.59.


The final model is not extremely strong, but its adequate F1 score ensures that it balances precision and recall to predict bank exits fairly well. It should provide a good basis for identifying customers that might be on the verge of leaving. Beta Bank should reach out and offer enticements to those customers to reduce churn.
